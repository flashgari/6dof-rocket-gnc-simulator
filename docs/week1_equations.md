# Week 1 Equations of Motion

The Week 1 model establishes the rigid-body dynamics foundation used by every later controller and verification case. The state is:

```text
x = [r_I, v_I, q_BI, omega_B]
```

where `r_I` and `v_I` are inertial position and velocity, `q_BI` maps body-frame vectors into the inertial frame, and `omega_B` is body angular velocity.

## Quaternion Attitude Representation

The simulator uses quaternions because the disturbed cases intentionally pass through large attitude excursions and inverted flight. Euler-angle propagation would introduce representation singularities and sequence-dependent ambiguity near large rotations. The quaternion kinematics are:

```text
q_dot_BI = 0.5 q_BI otimes [0, omega_B]
```

The unit-norm constraint is not cosmetic. A non-unit quaternion no longer represents an orthonormal rotation, so body-to-inertial force projection would become nonphysical. The quaternion norm check therefore verifies the numerical integrity of the attitude state, which later determines thrust direction, aerodynamic angle of attack, and CP/CM moment direction.

## Translational Dynamics

The inertial translational equations are:

```text
r_dot_I = v_I
m v_dot_I = R_BI(q) F_B + [0, 0, -mg]
```

Thrust and aerodynamic forces are modeled in body coordinates because they are tied to vehicle geometry. Gravity is naturally inertial. The quaternion-derived rotation `R_BI(q)` maps body-frame forces into the inertial frame before applying Newton's second law.

The key ascent coupling is thrust-vector projection. If the body `+z` axis is tilted by angle `theta` from inertial up, then:

```text
T_vertical = T cos(theta)
T_lateral  = T sin(theta)
```

Week 1 uses `theta = 0`, so all thrust contributes to vertical acceleration. Later failures occur when disturbance moments increase `theta`, reducing vertical thrust authority and increasing lateral acceleration.

## Rotational Dynamics

The body-frame rotational dynamics use Euler's rigid-body equation:

```text
I omega_dot_B + omega_B x (I omega_B) = tau_B
omega_dot_B = I^-1 [tau_B - omega_B x (I omega_B)]
```

The gyroscopic term is essential for a real rigid body because angular momentum is conserved in inertial space, while body-frame components rotate with the vehicle. Even if the Week 1 nominal case has zero angular rate, the term must be correct before adding off-axis torques, TVC moments, or inertial cross-coupling.

Applied moments are generated by force lever arms:

```text
tau = r x F
```

This same relationship later appears in both aerodynamic CP/CM moments and thrust-vector-control torque generation.

## Baseline Verification Case

For Week 1:

```text
m = 50 kg
T = 850 N
g = 9.80665 m/s^2
Cd = 0
q(0) = [1, 0, 0, 0]
omega_B(0) = [0, 0, 0] rad/s
```

The vertical acceleration is analytically:

```text
a_z = T / m - g = 7.19335 m/s^2
```

Therefore:

```text
z(10) = 0.5 a_z t^2 = 359.6675 m
v_z(10) = a_z t = 71.9335 m/s
```

This is a verification case, not only a demonstration. Matching the closed-form solution checks translational force balance, gravity sign convention, thrust projection, integration accuracy, and state-output consistency.

## Physical Meaning Of The Week 1 Plot

The altitude curve is parabolic because the net axial acceleration is constant:

```text
z(t) = z_0 + v_0 t + 0.5 (T/m - g) t^2
```

The vertical velocity curve is linear:

```text
v_z(t) = (T/m - g) t
```

The quaternion norm should remain at `1` to numerical precision. A drifting norm would imply that later force projections and attitude-error calculations were being computed with an invalid rotation operator. The Week 1 plot therefore verifies translational dynamics and attitude-state health before adding aerodynamic or control complexity.
